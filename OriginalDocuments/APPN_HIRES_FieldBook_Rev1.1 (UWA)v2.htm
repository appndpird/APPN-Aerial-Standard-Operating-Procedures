This fieldbook provides a standardised operational guide for APPN Hi￼Res UAV deployments using the Phase One iXM￼GS120 120 MP RGB camera mounted on a DJI M350 RTK. It is intended for trained APPN staff conducting high￼resolution RGB data acquisition for crop emergence, canopy architecture, and structural phenotyping applications. The protocol supports safe flight operations, consistent sensor configuration, and high￼quality, repeatable data capture across APPN field nodes.

 

This protocol must be followed for all standard APPN Hi￼Res UAV flights. For any flights that fall outside standard operating procedures, detailed records must be kept documenting all deviations and their implications for data quality and downstream analysis.

FIRST USE AND PRESEASON PREPERATION

Performance of the camera relies on correct installation of the PhaseOne camera within the gimbal, a perfectly balanced gimbal, and the drone performs best when the camera weight and centre of gravity has been factored into the DJI flight control software.

Please refer to the “NON ROUTINE, FIRST TIME, AND COMMISSIONING PROCEDURES” instruction at the end of the document, for information on installing the damper plate, gimbal installation and setup, gimbal balancing and drone centre of gravity balancing.

NOTE: Once you have set the gimbal balance, note the settings and offsets and take photos of the setting locations to ensure that you have reference settings for a stable gimbal. If the camera and gimbal goes out of balance, immediately inspect the balance settings, and set back to you known default balanced values for that camera.

 

PREFLIGHT PLANNING – M350

Ensure that you apply for UAV flight approvals for locations and dates of flights well in advance.  Further planning documentation for the Phase One payload on the M350 can be found here. Note also that before first flight with this sensor, there are several steps that need to be completed that are documented at the end of this protocol.

1.     Using a GPS survey system (Emlid, Trimble…) or a GIS software create a polygon of the area of interest. Make sure your polygon includes the areas where you will place your GCPs, with an additional 5m buffer to avoid incomplete data.

2.     Create a polygon of the area of interest.

3.     Save the polygon as a KML file using the convention: YYYYMMDD_XXXX (where YYYY = year; MM = Month; DD = Day (must be 2 digits), and XXXX is a short name reference or abbreviation for the job).

4.     Load this KML file onto a USB stick or microSD card for transfer to M350 controller.

5.     Power on controller.

6.     Open DJI Pilot 2 and select Flight Route → + Import Route (KMZ/KML). Alternatively, you can create an Area Route on the controller (such as by walking the field). Note that you do not need the UAV powered on for this, just the controller.

7.     Confirm that the Phase One iXM‑GS120 payload is correctly detected in DJI Pilot 2 [NC1] and selected as the active payload for the mapping mission. Ensure the correct aircraft model (M350/M300 RTK) is selected, and configure a custom camera profile using the appropriate Phase One P3 (80mm) parameters.

8.      

 

9.     Set the flight altitude mode to Relative to Take￼off Point.

10.  Set image overlap and triggering parameters in the Advanced Settings of the mapping mission as follows:

·       Front (forward) overlap: 80%[NC2] 

·       Side (lateral) overlap: 80%

·       Photo mode: Distance Interval Shot

11.  Set the course angle to align with the experimental layout:

·       Use row aligned orientation for plot-based trials where crop rows are clearly defined.

·       Use North-South orientation for broad scale paddock surveys.

12.  Set the flight altitude mode to Relative to Take￼off Point.[NC3] 

13.  The selected flight altitude and associated ground sampling distance (GSD), flight speed according to the mission standard type settings below. You must setect settings within the APPN HiRES Mission Standard Type Settings (Table 1), as these are optoimised to balance spatial resolution, coverage, and motion blur risk. The chosen altitude must be documented in the flight log.[NC4] [CC5]

ONSITE PREFLIGHT OPERATIONS – M350

1.     Conduct airspace and weather checks.

·       Check NOTAM (find in ERSA).

·       Ideally, no cloud cover. Avoid rapidly changing cloud conditions where possible.

·       Maximum wind speed for quality capture is 6m/s or 22km/hr. Any windspeed over 5m/s or 18km/hr should be recorded. [NC13] 

·       Do not operate in conditions below 0°C or above 40°C.

2.     Turn on aeronautical radio and set to local CTAF (find in ERSA).

3.     Set up DJI RTK 2 base station on tripod and power on at least 15 minutes prior to flight. Ensure it is set up in operation mode 5 (five LED blinks for communication with M350.

For any missions returning to the same location or requiring alignment to non-DJI data, the DJi D-RTK-2 base station must be set-up over a known point, that is surveyed in against a a GNSS base station(E.g. using AUSCORS or AUSPOS), or set to the location that was used for a previous survey. [NC14] Enter into the M350 COntroller and RTK settings. Manually edit the latitude and longitude of the best station, to set to the known surveyed (or past) location. The Aircraft should be switched off during this step.

4.     Deploy GCPs in the field ensuring good coverage of the area of interest (see diagram below). If using Propellor Aeropoints, ensure they are switched on and logging. If using standard GCPs, log GNSS coordinates using a Trimble or Emlid GNSS system.

5.     Setup landing pad and UAV in a safe RTH location.

·       In dusty environments an additional tarp should be used under the landing pad

6.     Attach Hi-Res payload to UAV:

·       Check that the lens and sensors are clean. If not, use professional lens cleaning wipes to clean them.

·       Check damping plate to ensure it is mounted securely to UAV and that USB-C cable is connected.

·       Remove the protective metal cap from the quick release receptacle.

·       Insert the gimbal into the quick release receptacle on the damping plate, while aligning the unlocked symbol on the bayonet opposite the white mark on the receptacle.

·       Rotate the non-slip ring to the right until the locked symbol is opposite the white mark on the receptacle and the bayonet clicks and locks. Note: This mount is tool‑less, but correct alignment is critical.

·      Remove lens cap.

·       Ensure CF express card is installed and that there is adequate space to store mission data.

7.     Power on radio controller; check battery status.

8.     Launch DJI Pilot 2.

9.     Load and review flight plan; checking operational height and double-checking area of interest, GCPs and reflectance panels are all within capture polygon.

10.  Power on aircraft; confirm connection to RC/GCS and battery status.

11.  On the controller, set up a safe UAV RTH location, RTH altitude and other geo-fencing settings. Check that mission is available on the controller.

12.  If you are using RTK, check that you are receiving GNSS corrections. There should be more than >8 satellites for good correction. Check the RTK resolution in the M350 controller settings, including that the solution is “FIX”, and the horizontal and vertical error is less than 0.03 (3cm).

13.   

 

 

FLIGHT OPERATIONS

1.     Ensure aircraft is in Position flight mode.[DS15] 

2.     Clear people/objects away from UAV.

3.     Notify crew/observers that takeoff is beginning.

4.     Begin manual takeoff, fly quickly to ~12m AGL and check stick controls work.

5.     Enable autonomous mission prepared earlier.

6.     Monitor UAV battery voltage/percentage in flight.

7.     After mission, switch back to Position mode to regain manual control.

8.     Land and disarm UAV.

9.     Leave UAV, transmitter, and sensor powered on; begin post-flight checks.

 

IN-FLIGHT DATA CONFIRMATION

1.     On RC Plus controller check image count and remaining storage on card (to ensure card did not fill up during mission). Review the images as the mission proceeds. Ensure that the image count is increasing as the images are written to the card. Listen for the “click” noise as images are captured. Review the histogram settings and the photo, to ensure that images have appropriate illumination.

 

 

POST-FLIGHT

1.     Power off drone then controller.

2.     Inspect drone for any damage and report for maintenance if necessary. Feel both batteries for excess or differential temperature. Feel the gimbal motors for excess or differential temperature. Inspect propellors, airframe and landing gear.

3.     Remove aircraft batteries.

4.     Remove payload, place in case.

5.     Pack up drone and all gear from site double checking against checklist at the start of this document to ensure everything is accounted for.

 

OFFLOAD DATA

2.     Remove CFExpress card from camera and offload data to computer using card reader.

3.     After offload and backup, format card for next mission.[NC16]

Motion Blur = Shutter Interval (s) × Drone Speed (m/s). Rule: Motion Blur ≤ GSD.

-          Motion blur: 0.00025 × 4 = 0.001 m = 1.0 mm

-          GSD at 50 m: 2.1 mm = 0.0021 m

2. Phase One iXM-GS120 (80 mm AF Lens) — Camera Capture Settings

Exposure Mode

Manual (Fixed Aperture + Fixed Shutter + Auto ISO)

Lock shutter speeds (1/2000 or faster – see table below), set aperture to f/8, let ISO adjust. This shutter and aperture priority ensures motion-blur ceiling is protected first, radiometric quality second.

EV Compensation

0 baseline — adjust to +1 in low-light / emergence conditions

+1 EV pushes the histogram right for more tonal information. Verify with live histogram.

ISO

Base 200 (sensor native) - max 3200

iXM-GS120 native ISO range is 200–6400 (RGB). Noise becomes visibly detectable above ISO 3200.

Aperture

f/8 min (sweet spot) — f/11 max

 

Sharpness peaks ~f/8 across the frame; edge softness appears wide of f/5.6 and diffraction softens past f/11.

Shutter Speed

Set per Shutter Speed Lookup (Table 3) — from 1/2000 to 1/16,000

 

Motion blur rule: shutter × speed ≤ GSD × 0.5 (half-pixel tolerance). See table below

Focus Mode

Manual focus locked to infinity (> 3 m) OR AF (single acquisition at mission start)

A calibrated fixed-focus 80 mm (infinity-only) achieves ±2 px RMSE vs. higher residuals on an uncalibrated AF. For research-critical work, consider requesting the fixed 80 mm (Phase One team recommendation)

Focus Bracketing

OFF

Bracketing multiplies frame count and breaks consistent GSD assumptions for ortho processing.

White Balance

N/A. It’s fixed to Daylight (5500K)

Fixed WB ensures radiometric consistency for post-processing

File Format

IIQ (RAW, 14-bit)

Expect ~100 MB per IIQ frame.

Environmental Capture Window

Cloud Cover (preferred)

Clear blue sky OR uniform high-altitude cover (cirrus / cirrostratus)

Both yield consistent illumination across the mission. Uniform altostratus is acceptable if genuinely even.

Cloud Cover (avoid)

Low-level broken / patchy cumulus

Hard shadows move across the AOI during capture → per-frame radiometric inconsistency.

Sun Angle and Capture Window

Ensure sun is ±20°of solar noon, approximately 2 hours before or after noon

Solar noon will depend upon time of year and latitude so please check here if unsure

Wind

≤ manufacturer limit for M350 RTK (12 m/s sustained); prefer ≤ 8 m/s for photogrammetric quality

Gust-induced position jitter degrades overlap consistency

GCPs, Boresight & Calibration

Ground Control Points

Minimum 5 per site (4 corners + 1 centre)

Surveyed with RTK/PPK receiver, ≤ 2 cm accuracy

Required for absolute accuracy validation. For repeat monitoring plots, install permanent GCP markers.

Cross-Runs over GCPs

INCLUDE — fly a perpendicular cross-strip over the GCP line after the main mission

Per Phase One ANZ guidance, cross-runs are valuable for boresight angle determination, especially after CoG calibration and with gimbal aligned to nadir. Adds ~5 min per mission.

Gimbal Calibration

Verify nadir alignment before each flight day

Even small gimbal offsets propagate into exterior orientation errors across a full mission.
